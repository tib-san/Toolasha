## Documentation Organization

### Root Level

Core documentation that should be read first:

- Project overview (README)
- Developer guide (AGENTS)
- Feature list (FEATURES)
- Version history (CHANGELOG)

### docs/planning/

Active planning documents for features currently in development or being considered.

### docs/reference/

Technical reference materials including API documentation and code indexes.

### docs/archive/

Historical documentation including completed work and superseded proposals. Kept for reference but not actively maintained.

- **completed/** - Implementation summaries for finished features
- **investigations/** - Research and analysis documents
- **proposals/** - Old feature proposals that were superseded or declined
